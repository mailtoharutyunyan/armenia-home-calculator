# Калькулятор строительства дома в Армении — дизайн-спецификация

Дата: 2026-07-19

## 1. Цель и объём

Веб-калькулятор, который по параметрам дома считает **объёмы материалов** и
**ориентировочную стоимость** строительства в Армении — отдельно для стадии
**«акт / коробка»** и **«под ключ»**. Поддерживает 4 конструктивные системы,
несколько типов фундамента, проверку по инженерным нормам, чек-лист получения
разрешения и редактируемый прайс материалов.

Интерфейс: **русский + армянский** (переключатель). Развёртывание: **GitHub Pages**
(статичная сборка, без сервера).

### Что это НЕ делает (честные оговорки, встроены в UI)

- **Не заменяет проект лицензированного инженера-конструктора.** Расчёт объёмов и
  проверка норм — инженерные эвристики (правила «рулём»), а не поверочный расчёт.
- **Не выдаёт разрешение на строительство.** Формирует только справочный чек-лист
  документов и шагов для подачи в госорганы РА.
- **Цены — стартовые и требуют актуализации.** Заложены реалистичные значения
  (ориентир — справочники Минфина РА minfin.am и рыночные данные), пользователь
  правит их на сайте; правки хранятся в `localStorage`.
- Итог — **диапазон «от–до»**, а не одно «точное» число.

## 2. Технологический стек

- **React + Vite + TypeScript**, сборка в статику → GitHub Pages.
- Состояние: **Zustand** (лёгкий стор для параметров дома, прайса, языка).
- Стили/UI: через **frontend-design skill** (Tailwind + собственная дизайн-система).
- i18n: словари JSON (`ru`, `hy`), простой хук `useT()`.
- Тесты: **Vitest** для движков (чистые функции).
- Экспорт: печать/PDF через `window.print()` + печатные стили; **DXF** — библиотека
  генерации DXF в браузере (напр. `dxf-writer`) для открытия плана в AutoCAD/LibreCAD.
- 2D-план: SVG (react-three-fiber/3D сознательно НЕ используем в v1).
- Хранение прайса: сид из `prices.json`, переопределения в `localStorage`,
  кнопка «сбросить к стартовым».

## 3. Архитектура: модули с одной ответственностью

```
src/
  data/
    prices.json          # стартовый прайс материалов и работ
    coefficients.json    # инженерные коэффициенты (толщины, армирование, факторы)
    permits.ts           # чек-лист разрешения РА (ru/hy)
    i18n/{ru,hy}.json     # словари интерфейса
  model/
    house.ts             # тип HouseParams + значения по умолчанию
    catalog.ts           # типы каталога цен, ед. измерения
  engine/
    quantities.ts        # HouseParams -> объёмы материалов (чистые функции)
    pricing.ts           # объёмы + цены -> смета (материалы+работа, стадии, диапазон)
    norms.ts             # HouseParams+объёмы -> список предупреждений
    compare.ts           # смета по каждой конструктивной системе для сравнения
  store/
    useProject.ts        # Zustand: параметры, прайс(с localStorage), язык
  ui/                    # компоненты (визард, смета, редактор цен, панель норм)
  App.tsx, main.tsx
```

### Поток данных

```
HouseParams ──> quantities() ──> Quantities
                                    │
              norms(params, quantities) ──> Warning[]
                                    │
              pricing(quantities, prices, finishLevel) ──> Estimate {акт, подключ, диапазон}
```

Пересчёт реактивный: любое изменение параметра/цены → мгновенный перерасчёт.

## 4. Модель данных

### 4.1 HouseParams

```ts
type ConstructionSystem = 'frame' | 'tuff' | 'aerated' | 'brick';
// frame = монолитный каркас + заполнение; tuff = несущий туф;
// aerated = несущий газоблок; brick = несущий кирпич

type FoundationType = 'strip' | 'slab' | 'pile' | 'column';
// ленточный | монолитная плита | свайный | столбчатый

interface HouseParams {
  system: ConstructionSystem;
  infillMaterial?: 'tuff' | 'aerated' | 'brick'; // только для frame
  length: number;          // м, габарит по внешним осям
  width: number;           // м
  floors: number;          // надземных этажей
  floorHeight: number;     // м, высота этажа (default 3.0)
  wallThickness: number;   // м (default по системе)
  foundation: FoundationType;
  basement: boolean;
  basementDepth?: number;  // м (default 2.4)
  roof: 'flat' | 'pitched';
  roofPitchDeg?: number;   // угол ската (для фронтонов), default 30
  windowAreaTotal: number; // м², суммарная площадь окон
  vitrageShare: number;    // 0..1 доля витражных окон
  exteriorDoors: number;   // шт
  interiorDoors: number;   // шт (или авто-оценка)
  finishLevel: 'economy' | 'standard' | 'premium';
  currency: 'AMD' | 'USD';
  region: RegionKey;       // регион РА -> сейсмобалл + глубина промерзания
  vatIncluded: boolean;    // показывать цены с НДС 20% или без
}

type RegionKey = 'yerevan' | 'gyumri' | 'vanadzor' | 'sevan' | 'aparan'
               | 'ararat_valley' | 'syunik' | 'other';
// у каждого региона: сейсмическая интенсивность (8/9 баллов) и
// нормативная глубина промерзания (см. coefficients.json)
```

Примечание: `region` заменяет фиксированную глубину промерзания и задаёт
сейсмический балл. Вся территория РА сейсмоопасна (8–9 баллов) — это влияет
на нормы и на обязательные сейсмоэлементы кладки.

### 4.2 Каталог цен (`prices.json`)

Каждая позиция хранит **вилку цен** (не одно «идеальное» число), собранную из
нескольких источников, плюс работу:
```ts
interface PriceItem {
  key: string;             // 'concrete_b25', 'rebar_a500', 'tuff_block', ...
  labelRu: string; labelHy: string;
  unit: string;            // 'м³', 'т', 'шт', 'м²', 'компл'
  materialMin: number;     // минимальная найденная цена (эконом-поставщик)
  materialTypical: number; // типичная/рыночная
  materialMax: number;     // высокая/премиум
  labor: number;           // цена работы за ед. (AMD), 0 если не применимо
  sources: string[];       // ссылки/названия поставщиков, откуда взята вилка
  note?: string;           // напр. «цена без доставки/НДС», «list.am — проверить вручную»
}
```

**Глобальный режим цен** (переключатель в смете): «эконом» (использует `materialMin`) /
«рыночная» (`materialTypical`) / «премиум» (`materialMax`). Итоговый **диапазон «от–до»**
строится напрямую из `min…max` по позициям — то есть разброс отражает реальную
разницу поставщиков, а не выдуманный ±%. Любую цену можно переопределить вручную.
Явная пометка: **ни один источник не является эталоном; проверяйте актуальные цены**.

Разделы каталога: **бетон** (единый ряд класс↔марка: B15/M200, B20/M250, B22.5/M300,
B25/M350, B30/M400),
**арматура** (A400, A500 — усреднённо по массе), **туф**, **газоблок**, **кирпич**,
**плиты перекрытия ПК** и **монолит**, **цемент/песок/щебень**, **гидроизоляция**,
**утеплитель**, **окна обычные** (м²), **окна витражные** (м²), **двери входные/межкомнатные**,
**штукатурка/шпаклёвка/покраска** (м²), **напольные покрытия** (м²), **кровля**
(плоская/скатная, м²), **фасад** (м²), **электрика/сантехника/отопление** (м²),
**земляные работы** (м³, выемка/засыпка), **песок/щебень подушки**, **отмостка** (м²),
**лестница** (компл/м³), **клей для газоблока** (кг/м³).

Курс AMD↔USD — редактируемое поле (default ~ 385 драм/USD, помечено «проверьте курс»).

**Стартовые цены (из реального проекта my-home, драмы, без НДС — редактируемы).**
Приводятся как `typical`; `min`/`max` разворачиваются как typical ×0.9 / ×1.15
(редактируются вручную по мере сбора цен поставщиков):
- Бетон B15/M200 29 000; B20/M250 30 000; B22.5/M300 32 000; B25/M350 34 000;
  B30/M400 ≈ 36 000 /м³; подбетонка М100 30 000/м³; бетононасос +3 000/м³.
- Арматура А500С 350 000/т (Ø8 146.62, Ø10 215.56, Ø12 300.72 драм/м).
- Газоблок 42 000/м³; щебень/песок подсыпка 8 000/м³.
- Работа (бригада, коробка) ≈ 11 000/м² площади.
- Клей/цемент/проволока/прочее — комплектом (уточняется).

**Источники цен (ссылки в редакторе):** MM Leader (бетон) `mmlider.am`,
Stalmetural `stalmetural.am`, Met-Trans `met-trans.am` (арматура),
List.am `list.am` (газоблок), справочники Минфина РА `minfin.am`. Пометка: цены на
List.am требуют авторизации и часто меняются — проверять вручную.

### 4.3 Инженерные коэффициенты (`coefficients.json`, редактируются в «Расширенно»)

- Толщины по умолчанию: лента 0.4×0.8 м, плита 0.3 м, стены (туф 0.4, газоблок 0.3,
  кирпич 0.38, каркас-заполнение 0.3), **перекрытие монолит 0.17 м** (было 0.2),
  стяжка 0.05 м, армопояс 0.3×0.2 м, столбчатый фундамент высота столба 1.5 м.
- **Армирование (кг/м³) — единый ряд по элементам:** лента/ростверк 80, плита 100,
  сваи/столбы 90, колонны 170 (сейсмика; было 120), перекрытие 110, армопояс 100,
  сейсмо-сердечники 150, перемычки 120, стены подвала 90.
- Скатная кровля: множитель площади 1.3; каркас колонн: шаг сетки 4 м, сечение 0.4×0.4 м.
- Отходы/запас на резку: +5 % к штучным/погонным материалам.
- **Раствор кладки:** туф/кирпич ≈ 0.2×V_кладки; **газоблок — клей ≈ 0.025×V**.
- Уровень отделки — множители к отделочным разделам: economy ×0.8, standard ×1.0,
  premium ×1.6.
- **Внутренние стены/перегородки:** длина внутренних несущих ≈ 0.5×периметр;
  перегородки (для «под ключ») ≈ 0.8×периметр, толщина 0.1 м.
- **Земляные работы:** выемка под ленту/плиту/подвал (м³), обратная засыпка 0.6×выемки,
  песчано-щебёночная подушка 0.1 м под фундаментом, подбетонка (тощий бетон) 0.05 м.
- **Отмостка:** ширина 1.0 м по периметру; **гидроизоляция фундамента/цоколя** по площади.
- **Сейсмо-элементы** (для несущей кладки туф/кирпич): вертикальные ж/б сердечники
  в углах и на пересечениях + по длине стен с шагом ≤ 3 м, сечение 0.25×0.25 м на
  всю высоту; армопояс обязателен на каждом этаже.
- **Промерзание по региону (м):** Ереван/Араратская долина 0.8, Ванадзор 1.2,
  Гюмри 1.5, Севан/Апаран 1.8, Сюник 1.4, прочее 1.2.
- **Сейсмобалл по региону:** Ереван/Араратская долина/Сюник — 8, Гюмри/Ванадзор/
  Севан/Апаран — 9 (значения редактируемы).
- **НДС:** 20 % (переключатель «с/без НДС»). **Доставка/логистика:** +5 % к материалам
  (настраивается; больше для удалённых регионов).
- **Диапазон «от–до»** строится ТОЛЬКО из вилки цен поставщиков: нижняя граница по
  `materialMin`, верхняя — по `materialMax` (никакого дополнительного ±%).
- **Резерв на непредвиденное: +12 %** (настраивается 10–15 %) — применяется к обеим
  границам одинаково. В UI подписывается «включён резерв 12 %».
- Факторы работы: где `labor` не задан по позиции, работа = материал × labor_factor
  раздела (значения по умолчанию в файле).

## 5. Движок объёмов (`quantities.ts`)

Обозначения: `A = length×width` (пятно застройки), `P = 2(length+width)` (периметр),
`H = floors×floorHeight` (высота надземной части).

Общая длина осей несущих стен `L_нес = 1.5×P` (наружные + внутренние — см. «Стены»).

**Земляные работы** (для всех типов): выемка грунта под фундамент/подвал (м³),
обратная засыпка = 0.6×выемки, песчано-щебёночная подушка `A_подошвы × 0.1`,
подбетонка (тощий бетон) `A_подошвы × 0.05`. Для ленты `A_подошвы = L_нес × w_лента`;
для плиты `A_подошвы = A`.

**Фундамент** (по типу; лента/сваи/столбы идут под ВСЕМИ несущими осями `L_нес`):
- `strip`: `V_бетон = L_нес × w_лента × h_лента`; арматура = V×80.
- `slab`: `V = A × t_плиты`; арматура = V×100.
- `pile`: сваи с шагом 2 м вдоль несущих осей: `n = L_нес/2`; `V_свай = n×(π/4)d²×L_сваи`
  (d=0.3, L=3) + ростверк как облегчённая лента по `L_нес`; арматура = V×90.
- `column`: столбы с шагом 2 м вдоль несущих осей: `n = L_нес/2`;
  `V = n×0.4×0.4×h_столба` (h=1.5); арматура = V×90.

**Гидроизоляция фундамента/цоколя** — по площади вертикальных поверхностей.
**Отмостка** — по периметру, ширина 1.0 м.

**Подвал** (если есть): монолитные ж/б стены толщиной `t_подвала = 0.3 м`:
`V = P × 0.3 × basementDepth`, арматура ×90; выемка грунта учтена в «Земляные работы».

**Стены** (надземные):
- Длина наружных стен = `P`; **внутренние несущие** ≈ `0.5×P`; итого несущих `L_нес = 1.5×P`.
- Площадь брутто `S = L_нес×H`, за вычетом проёмов `S_пр = windowAreaTotal + doorsArea`.
  `S_нетто = S − S_пр`.
- `tuff/aerated/brick` (несущие): `V_кладки = S_нетто × wallThickness` **минус объём
  армопояса и сейсмо-сердечников** (чтобы не считать дважды);
  кол-во блоков = V/объём_блока (+5 % запас); раствор: туф/кирпич 0.2×V,
  газоблок — клей 0.025×V.
- `frame`: колонны (`n` по сетке 4 м, сечение 0.4×0.4, высота H) + обвязочные балки
  по периметру и внутренним осям на каждом этаже (`V ≈ L_нес×0.4×0.4`) + заполнение
  из `infillMaterial` (как кладка, ненесущая толщина).
- **Армопояс** (несущая кладка) на каждом этаже: `V = L_нес×0.3×0.2`, арматура ×100.
- **Сейсмо-сердечники** (несущая кладка туф/кирпич): количество = углы + пересечения
  + по стенам с шагом ≤3 м; `V = n×0.25×0.25×H`, арматура ×150.
- **Перемычки** над проёмами: суммарная длина ≈ (кол-во окон+дверей)×(ширина+0.5),
  сечение 0.25×0.2, арматура ×120.
- **Перегородки** (для «под ключ»): `V = 0.8×P × H × 0.1` из **газоблока/пазогребня**
  (толщина 0.1 м, независимо от несущего материала), штукатурка с двух сторон.

**Перекрытия**: на каждый надземный этаж потолок площадью `A`.
- монолит: `V = A × t × floors` (t=0.17); арматура ×110.
- сборные ПК: количество плит = A/площадь_плиты × floors.

**Лестница** (при `floors ≥ 2`): монолитный марш на каждый межэтажный переход
`n = floors−1`; объём бетона `≈ n×2.5 м³`, арматура ×110; отделка ступеней в «под ключ».

**Стяжка**: `A × floors × t_стяжки` → цемент/песок.

**Штукатурка/шпаклёвка** (под ключ): наружные стены (снаружи+внутри) + перегородки
(с двух сторон) → суммарная площадь с учётом внутренних стен и перегородок.

**Кровля**: плоская `≈ A` (гидроизоляция+утеплитель+стяжка); скатная `A×1.3`
(стропила+покрытие+утеплитель). При скатной — **фронтоны**: `≈ width²×tan(pitch)/4 × 2`
из материала стен.

**Окна**: `windowAreaTotal`, из них витраж = `× vitrageShare`, остальное — обычные.
**Двери**: входные (шт), межкомнатные (шт; если не заданы — авто ≈ floors×(A/20)).

**Инженерка**: электрика/сантехника/отопление считаются по общей площади
`A_общ = A × floors` с ценой за м².

**Отделка**: полы = `A_общ`; стены/потолки — по площадям выше; уровень отделки — множитель.

Выход: объект `Quantities` с полями по разделам (значения + единицы).

## 6. Движок стоимости (`pricing.ts`)

- Каждый объём × (`materialMin|Typical|Max` по режиму цен + `labor`) из каталога →
  строка сметы (раздел, материал, работа, сумма).
- **Стадии**: «акт/коробка» = фундамент + стены/каркас + перекрытия + армопояс + кровля
  (базовая) + чёрновая стяжка. «Под ключ» = «акт» + окна/двери + отделка + инженерка + фасад.
- Множитель уровня отделки применяется к отделочным разделам под-ключ.
- **Доставка/логистика**: +% к материалам (по региону).
- **НДС 20 %**: если `vatIncluded` — цены показываются с НДС; иначе отдельной строкой.
- Итог: сумма → **доставка** → **резерв +12 %** → **цена за м²** (`итог / A_общ`).
  **Диапазон «от–до»** = пересчёт итога по `materialMin` и `materialMax` (вилка
  поставщиков), а не искусственный ±%. Разбивка «материалы» vs «работа».
  Валюта AMD/USD по курсу.

## 7. Сравнение технологий (`compare.ts`)

Прогоняет те же `HouseParams` через `quantities+pricing` для каждой из 4 систем
(подставляя типичные толщины) → таблица «стоимость акт / под ключ / цена за м²»
по колонкам туф | газоблок | каркас | кирпич.

**Несущий газоблок в сейсмозоне запрещён нормой** (§8): колонка показывается с бейджем
🔴 «не допускается как несущий в РА», а её расчётный вариант заменяется на
**каркас + газоблок-заполнение** (корректная для РА схема) — чтобы сравнение
оставалось полезным, но не подсказывало запрещённое решение.

## 8. Валидатор норм (`norms.ts`)

Возвращает `Warning[]` с уровнями `error` 🔴 / `warning` 🟡 / `info` ℹ️ и текстом (ru/hy).
Каждое правило ссылается на **конкретный армянский норматив (ՀՀՇՆ)** для доверия:

- **ՀՀՇՆ 30-01-2014** — градостроительство: отступ от границы участка ≥ 3 м;
  застройка участка ≤ 40 % (по зоне).
- **ՀՀՇՆ 31-01-2014** — жилые здания: упрощённая процедура ≤ 2 надземных + 1 подземный
  этаж и площадь ≤ 300 м²; высота жилой комнаты ≥ 2.7 м; коридор ≥ 2.1 м; мин. комната
  8 м²; естественное освещение окон ≈ ≥ 1/8 площади пола.
- **ՀՀՇՆ II-6.02** — сейсмический расчёт (требуется инженер-конструктор).
- **ՀՀՇՆ 31-02** — нормы для индивидуальных домов (ожидается в 2026).

Правила-проверки (пороги в `coefficients.json`, редактируемы):

- **🌍 Сейсмика (приоритет, вся РА — 8–9 баллов):**
  - **Несущий газоблок** в сейсмозоне → 🔴 (в РА как несущий материал не применяется;
    рекомендация — каркас с заполнением или туф с сейсмопоясами).
  - Несущая кладка `floors > 3` (при 9 баллах — `> 2`) → 🔴 (нужен каркасный расчёт).
  - Соотношение габаритов `длина/ширина > 2` → 🟡 (неблагоприятно для сейсмики).
  - Марка бетона несущих конструкций ниже **B25** в сейсмозоне → 🔴 (единственное
    правило по марке; вся РА — сейсмозона, поэтому отдельное правило «B20 при 3 этажах»
    не нужно).
  - ℹ️ Вертикальные сердечники + армопояс для несущей кладки движок добавляет
    автоматически (§5). В «Расширенно» есть галочка «без сейсмоусиления» — при её
    включении выдаётся 🔴 (нарушение) и элементы исключаются из объёмов.
- **Мин. армирование**: если пользователь вручную задал кг/м³ по элементу ниже порога → 🟡.
- **Толщина несущих стен**: туф `< 0.4` при `floors ≥ 2` → 🟡; кирпич `< 0.38` → 🟡.
- **Глубина фундамента**: заложение меньше глубины промерзания региона → 🟡.
- **Площадь остекления**: окна `> 40 %` площади стен → 🟡 (теплопотери/устойчивость).
- **Высота этажа**: жилая комната `< 2.7 м` (ՀՀՇՆ 31-01-2014) → 🟡; `> 3.5 м` → ℹ️.
- **Упрощённая процедура** (ՀՀՇՆ 31-01-2014): если `floors > 2` (надземных) или
  площадь застройки `> 300 м²` → ℹ️ «нужна полная процедура/экспертиза, не упрощённая».
- **Естественное освещение**: суммарная площадь окон `< 1/8` площади пола → 🟡.
- **Подвал**: `basementDepth > 2.5 м` → ℹ️ (спец. гидроизоляция/расчёт).
- **Валидация ввода**: 0 этажей, отрицательные/нулевые габариты → 🔴, блокирует расчёт.

Панель «Предупреждения» всегда видна над сметой; при наличии 🔴 итог помечается
«расчёт неполный/некорректный». Каждое предупреждение показывает **номер норматива
(ՀՀՇՆ)** и порог, по которому сработало — кликом раскрывается пояснение.

### 8.1 Справочник нормативов (виден на сайте, `data/norms-reference.ts`)

Отдельный **раздел «Нормативы РА»** в интерфейсе — полная таблица, доступная всегда
(не только при предупреждениях). Каждая строка (ru/hy):
`норматив | что регулирует | требование/порог | уровень при нарушении`.
Содержит все использованные стандарты и их значения:
- **ՀՀՇՆ 30-01-2014** — градостроительство: отступ ≥ 3 м, застройка ≤ 40 %.
- **ՀՀՇՆ 31-01-2014** — жилые здания: упрощённая процедура ≤ 2+1 этаж и ≤ 300 м²,
  высота комнаты ≥ 2.7 м, коридор ≥ 2.1 м, комната ≥ 8 м², освещение ≥ 1/8 площади пола.
- **ՀՀՇՆ II-6.02** — сейсмика (8–9 баллов): марка бетона ≥ B25, сердечники + армопояс,
  ограничение этажности несущей кладки, соотношение габаритов ≤ 2.
- **ՀՀՇՆ 31-02** — индивидуальные дома (ожидается 2026).
- Инженерные пороги: марка бетона по этажности, мин. армирование, толщина несущих
  стен, глубина заложения ≥ глубины промерзания региона, остекление ≤ 40 %.

Каждая строка справочника снабжена ссылкой на официальный источник (Минюст/Кадастр РА,
где опубликованы ՀՀՇՆ) и пометкой «ориентир, сверяйтесь с действующей редакцией».
Предупреждения в панели норм ссылаются на строку этого справочника (по номеру ՀՀՇՆ).

## 9. Чек-лист разрешения РА (`permits.ts`)

Справочный список (с крупным дисклеймером «не юридическая консультация; уточняйте
в муниципалитете / Комитете градостроительства»):
1. Правоустанавливающие документы на землю.
2. Архитектурно-планировочное задание (АПЗ) от муниципалитета.
3. Эскизный проект и его согласование.
4. Рабочий архитектурно-строительный проект (лицензированная организация).
5. Инженерно-геологические изыскания участка.
6. Экспертиза проекта (по этажности/площади при необходимости).
7. Разрешение на строительство (уполномоченный орган).
8. Технадзор и акт ввода в эксплуатацию по завершении.

Каждый пункт — краткое пояснение ru/hy. Список статичный, помечен как ориентир.
Отметка: при ≤ 2 надземных + 1 подземный этаж и площади ≤ 300 м² возможна **упрощённая
процедура** (ՀՀՇՆ 31-01-2014); иначе — полная с экспертизой.

### 9.1 Таймлайн этапов и сроки (`data/stages.ts`)

Горизонтальный таймлайн из 8 фаз с оценкой длительности (диапазон, зависит от площади;
масштабируется коэффициентом от `A_общ`). Значения-ориентиры, редактируемы:
1. Оформление документов и разрешение — 1–3 мес.
2. Земляные работы и фундамент — 2–4 нед.
3. Каркас/несущие стены и колонны — 3–6 нед.
4. Перекрытия и лестницы — 2–4 нед.
5. Кровля — 1–3 нед.
6. Окна, двери, фасад — 2–4 нед. (конец стадии «акт/коробка»).
7. Инженерные сети — 3–5 нед.
8. Отделка «под ключ» — 6–12 нед.
Итог: суммарный ориентировочный срок «до акта» и «под ключ» (диапазон).

## 9.2 2D-план и экспорт в CAD (`ui/Plan2D.tsx`, `engine/dxf.ts`)

Параметрическая **2D-схема** дома, синхронная с параметрами (реалтайм):
- Контур по габаритам `length×width`, наружные стены с реальной толщиной,
  внутренние несущие (по коэффициенту), проёмы окон и дверей, размерные линии,
  отдельный слой контура фундамента, переключение вида **по этажам**.
- Рендер — SVG (без сервера, масштабируется, печатается).

**Экспорт в CAD:** та же геометрия выгружается в **DXF** (открытый формат, открывается
в **AutoCAD / LibreCAD / nanoCAD**) через JS-библиотеку генерации DXF, плюс экспорт
плана в **PDF**. Это НЕ встроенный AutoCAD и НЕ авто-раскладка комнат — это точная
параметрическая схема и CAD-совместимый файл. Слои DXF: стены, проёмы, размеры,
фундамент.

## 10. Интерфейс (frontend-design skill)

- **Лендинг**: заголовок, суть, дисклеймеры, кнопка «Рассчитать».
- **Визард по шагам** (прогрессивное раскрытие; «Расширенные опции» для толщин/коэффициентов):
  1) Регион и конструктив/материал → 2) Габариты и этажи → 3) Фундамент и подвал →
  4) Крыша → 5) Окна и двери → 6) Уровень отделки и НДС.
- **2D-план** (SVG, реалтайм, по этажам) с кнопками **экспорт DXF** (AutoCAD) и PDF.
- **Итоговая смета**: диапазон «от–до» (из вилки цен поставщиков), цена/м², разбивка
  по разделам и стадиям (акт / под ключ), «материалы vs работа», строка резерва,
  переключатель **режима цен** (эконом/рыночная/премиум), НДС, экспорт PDF/печать.
- **Панель норм** с тремя уровнями предупреждений; каждое — с номером ՀՀՇՆ и порогом.
- **Раздел «Нормативы РА»** (§8.1): постоянно доступная таблица всех стандартов
  (ՀՀՇՆ 30-01-2014, 31-01-2014, II-6.02, 31-02) с требованиями, порогами и ссылками
  на официальные источники. Видна в меню сайта.
- **Сравнение технологий**: таблица 4 систем.
- **Таймлайн этапов** со сроками (до акта / под ключ).
- **Редактор цен**: таблица каталога (min/типичная/max), инлайн-правка, «сбросить»,
  **ссылки на поставщиков** (MM Leader, Stalmetural, met-trans, list.am, minfin).
- **Чек-лист разрешения**: раскрывающийся список (упрощённая/полная процедура).
- **Переключатель языка** (ru/hy) и валюты (AMD/USD).
- Подсказки-тултипы у полей (что входит в площадь, что такое «акт» и т.п.).

## 11. Обработка ошибок

- Валидация ввода на уровне формы (границы значений, блокировка при 🔴).
- Отсутствующая цена в каталоге → строка помечается «цена не задана», в итоге
  показывается предупреждение, а не 0.
- Повреждённый `localStorage` прайс → откат к сид-прайсу + уведомление.

## 12. Тестирование (Vitest)

- `quantities`: эталонные дома (10×10 2 этажа, лента; 8×12 1 этаж, плита; каркас 3 этажа)
  → объёмы в ожидаемых диапазонах.
- `pricing`: проверка стадий, множителя отделки, резерва, диапазона, цены/м².
- `norms`: срабатывание каждого правила на граничных значениях, включая **сейсмику**
  (несущий газоблок → 🔴; кладка выше допустимой этажности → 🔴; марка < B25 → 🔴).
- `compare`: 4 системы возвращают ненулевые сопоставимые сметы.
- `pricing` (вилка): режим эконом/рыночная/премиум даёт min ≤ typical ≤ max.
- `dxf`: экспорт валидного DXF с ожидаемыми слоями (стены/проёмы/размеры/фундамент).
- `stages`: сроки масштабируются с площадью, диапазон корректный.

## 14. Допущения (зафиксированы, редактируемы)

- Все коэффициенты и цены — стартовые ориентиры, требуют проверки инженером/актуализации.
- Оценка — концептуальная (early-stage), не тендерная смета.
- Межкомнатные двери и часть инженерки оцениваются по площади при отсутствии точного ввода.
- **Сейсмика:** расчёт учитывает сейсмичность РА (8–9 баллов) на уровне норм-эвристик
  и обязательных сейсмоэлементов (сердечники, армопояс). Это **не** сейсмический
  поверочный расчёт — его выполняет лицензированный инженер по ՀՀՇՆ/RABC.
